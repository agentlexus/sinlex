# ТЗ: модуль «Литьё» (кабинет Sinlex)

Версия: **2026-06-04**  
Статус: **внедрено (базовая версия 2026-06-04)**  
Связанный код (планируемый):
- `page_modules/7_Casting.py` (условное имя; финальный индекс уточняется)
- директория проектов: `casting/` (аналог `projects/`)
- переиспользуемые модули: `5_Upload.py`, `viewer_3d.py`, `extraction_tool/*`, `hybrid_analysis.py`, `pdf_analysis.py`

---

## 1. Цель

Добавить в личный кабинет Sinlex отдельный модуль **«Литьё»**:
- вести список литьевых проектов (аналог страницы «3D проекты»);
- загружать STEP-модели, считать по ним объём/массу/габариты через PythonOCC;
- учитывать особенности литейной технологии (тип литья, материал, усадка, припуски);
- считать **стоимость литья** по упрощённой, но параметризуемой формуле;
- подсвечивать **риски по минимальной толщине стенок** для выбранного типа литья.

## 2. Область работ

### 2.1. В scope

- Новый модуль Streamlit:
  - новый файл `page_modules/7_Casting.py` (название и индекс — предварительные);
  - пункт в навигации (в сайдбаре) «Литьё».
- Новая директория проектов:
  - `/opt/sinlex/casting/` — **аналог** `/opt/sinlex/projects/`;
  - структура проекта внутри папки — максимально похожая на 3D‑проекты (см. §4.2).
- UI:
  - список литьевых проектов с колонками **Название / Материал / Стоимость / Дата**;
  - кнопка **«Новое литьё»**, открывающая страницу нового проекта (см. §3.2);
  - страница проекта: **3D‑просмотр слева, параметры/формы справа**.
- Функциональность:
  - загрузка STEP‑файла (аплоудер) с теми же ограничениями, что и у 3D‑проектов;
  - расчёт геометрии (объём, масса, габариты, min толщина стенки) через существующий OCC‑pipeline;
  - ввод пользователем технологических параметров литья (тип, материал, усадка, припуски);
  - расчёт стоимости литья по формуле из §5;
  - предупреждения по минимальной толщине стенки для выбранного типа литья.

### 2.2. Вне scope

- Любые изменения тарификации, биллинга, лимитов проектов и оплат (используются существующие тарифы).
- Интеграция с внешними системами (ERP, MES).
- Автоматический подбор литейного процесса/сплава по базе производств.
- Визуальная подсветка проблемных зон стенок в 3D (пока только текстовые предупреждения).

## 3. UI и пользовательские сценарии

### 3.1. Страница «Литьё» (список проектов)

**URL/доступ:** аналог `3D проекты`, отдельный пункт меню «Литьё`.  
**Источник данных:** `casting/projects.json` + файловая структура `casting/<folder>/`.

Требования:
- Таблица по образцу 3D‑проектов:
  - Колонки: **Название**, **Материал**, **Стоимость**, **Дата**.
  - Выравнивание колонок — как в 3D‑проектах (Название слева, остальные по центру).
  - Строки кликабельные: клик по названию открывает страницу проекта.
- Кнопка **«Новое литьё»**:
  - расположение и стиль — аналог кнопки «Новый проект»;
  - при нажатии создаёт новый литьевой проект и переводит на его страницу (см. §3.2).
- Пустое состояние:
  - если проектов нет — показать сообщение «Пока нет литьевых проектов.».

### 3.2. Страница проекта «Литьё»

**Цель:** максимально повторить UX 3D‑проекта.

Компоненты:
- **Аплоудер STEP**:
  - тот же компонент и ограничения, что в модуле 3D‑проектов;
  - ограничения по размеру/количеству/типу файлов — полностью переиспользовать.
- **После успешной загрузки:**
  - слева — 3D‑viewer (тот же `viewer_3d` / pipeline pythonOCC);
  - справа — параметры и результаты:
    - базовые геометрические параметры (объём, масса, габариты, min толщина стенки);
    - блок параметров литья (см. §4.3);
    - рассчитанная стоимость литья;
    - предупреждения по толщине стенки.
- Кнопки/действия:
  - кнопка пересчёта стоимости (если требуется явный триггер);
  - автоматический пересчёт при изменении ключевых параметров (тип литья, материал, усадка, припуски).

## 4. Данные и структура проектов

### 4.1. Директория `/opt/sinlex/casting`

Создать:
- `/opt/sinlex/casting/` с правами `sinlex:sinlex`;
- внутри — одна папка на проект, по аналогии с `/opt/sinlex/projects/<folder>/`.

Требования к структуре проекта (минимум):
- `casting/<project_name>/`:
  - `source.step` — исходный загруженный STEP;
  - `meta.json` — метаданные проекта (название, материал, тип литья, даты и т.д.);
  - `analysis.json` / `geometry.json` — результаты OCC‑анализа (объём, габариты, min толщина и др.);
  - `costing.json` — параметры/результаты расчёта стоимости;
  - дополнительные артефакты по образцу 3D‑проектов (логи, GLB и т.п.) — переиспользовать существующую логику.

### 4.2. `casting/projects.json`

Аналог `projects/projects.json`:
- Структура записи проекта:
  - `name` — название проекта;
  - `material` — выбранный материал литья (человекочитаемый, напр. «Сталь», «Алюминий», «Чугун»);
  - `total_cost` — последняя рассчитанная стоимость литья;
  - `updated_at` / `created_at` — ISO‑даты для сортировки и отображения;
  - дополнительные поля по необходимости (тип литья, статус).
- Список на странице «Литьё» должен сортироваться по активности (аналог 3D‑проектов, по `updated_at`).

## 5. Параметры литья и формула стоимости

### 5.1. Пользовательские параметры

На странице проекта справа (в параметрах) добавить блок:

- **Тип литья** (обязательное поле, селект):
  - ЛПД (литьё под давлением),
  - ЛВМ (литьё по выплавляемым моделям),
  - ЛГМ (литьё по газифицируемым моделям),
  - Литьё в землю,
  - ЛК (литьё в кокиль).
- **Материал**:
  - Сталь,
  - Алюминий,
  - Чугун.
- **Усадка, %**:
  - числовое поле с плавающей точкой;
  - разумные пределы (например, 0–10 %; значения вне диапазона подсвечивать).
- **Припуск на сторону, мм**:
  - числовое поле, >= 0;
  - влияет на расчёт габаритов и объёма заготовки.

При изменении любого из этих параметров UI должен:
- пересчитывать **габариты заготовки** (с учётом усадки и припусков);
- пересчитывать **объём** и **массу отливки** / заготовки;
- пересчитывать **стоимость литья** (см. ниже);
- обновлять предупреждения по толщине стенки.

### 5.2. Геометрические величины (из OCC)

Использовать уже существующий OCC‑анализ:
- объём детали `V` (в м³ или см³, конвертация к массе через плотность);
- габариты (X, Y, Z);
- минимальная толщина стенки `t_min` (уже считается; если нет — добавить на базе существующего анализа);
- при необходимости — карта толщин (для будущего расширения, сейчас достаточно min значения).

### 5.3. Стоимость литья (первая версия)

В ТЗ зафиксировать модель:

- Обозначения:
  - `m` — масса отливки;
  - `P_cast` — цена сплава литья (за кг);
  - `P_alloy` — цена исходного сплава (за кг);
  - `K_отх` — коэффициент отходов (зависит от типа литья/материала; задаётся таблицей или константами);
  - `C_form` — фиксированная стоимость формовки (функция от объёма опоки и сложности);
  - `K_сложн` — коэффициент сложности (задаётся таблицей по типу литья и, при желании, по толщине/габаритам).

- Базовая формула:

> **Стоимость литья** =  
> `(масса_отливки * цена_сплава_литья) + (масса_отливки * K_отх * цена_сплава) + фиксированная_стоимость_формовки * коэфф_сложности`

В коде следует придерживаться нейтральных имён переменных (`m`, `P_cast`, `P_alloy`, `K_waste`, `C_form`, `K_complexity`), а в UI использовать русские подписи.

Требования:
- Значения `K_отх`, `C_form`, `K_сложн`, `P_cast`, `P_alloy` пока можно задать:
  - либо константами в коде (словарь по типу литья + материалу),
  - либо отдельным конфигом (`casting_config.json`) — на усмотрение реализации.
- Результат выводить:
  - в рублях, с форматированием как в 3D‑проектах (`1 234 ₽`);
  - как в списке проектов (колонка «Стоимость»), так и на карточке проекта.

## 6. Анализ геометрии и предупреждения по толщине стенки

### 6.1. Переиспользование OCC‑анализа

Требования:
- Не писать новый геометрический движок. Использовать тот же pipeline, что для 3D‑проектов:
  - загрузка STEP через pythonOCC;
  - расчёт объёма/габаритов;
  - расчёт минимальной толщины стенки (или доработать существующую функцию до явного поля `min_wall_thickness`).
- Результаты хранить в `analysis.json` (или аналогичном файле), а в UI подтягивать из кеша, чтобы:
  - не пересчитывать тяжелый OCC при каждом изменении только «тип литья / материал / усадка / припуски»;
  - геометрия считается один раз после загрузки/обновления STEP.

### 6.2. Правила предупреждений по толщине

Для каждого **типа литья** и **материала** задать минимально допустимую толщину `t_min_allowed` (таблица в коде или конфиг):
- Пример (может быть уточнён заказчиком):
  - ЛПД, алюминий: `t_min_allowed = 2.5 мм`;
  - ЛВМ, сталь: `t_min_allowed = 3.0 мм`;
  - ЛГМ, чугун: `t_min_allowed = 4.0 мм`;
  - и т.д.

Логика:
- Если `t_min` < `t_min_allowed` для выбранной комбинации:
  - показывать **предупреждение** (желательно `st.warning`):
    - текст в духе: «Минимальная толщина стенки X мм меньше рекомендуемой для выбранного типа литья (Y мм). Возможен брак / недолив.»
- Если данных по толщине нет (анализ не выполнен/ошибка):
  - показывать информационное сообщение о том, что контроль толщины недоступен.

## 7. API и внутренние функции

### 7.1. Создание/удаление литьевых проектов

Функциональность аналогична 3D‑проектам:
- создать директорию в `casting/`;
- инициализировать `projects.json` / `meta.json` по шаблону;
- обеспечить удаление проекта (иконка корзины в таблице «Литьё» с подтверждением).

### 7.2. Расчёт и сохранение результатов

- При успешной загрузке STEP:
  - запустить OCC‑анализ (если нет валидного кеша);
  - сохранить геометрию и min толщину;
  - по текущим параметрам литья рассчитать стоимость и сохранить её в проект.
- При изменении параметров литья:
  - использовать уже сохранённые геометрические величины;
  - пересчитывать только часть, зависящую от параметров и формулы стоимости;
  - обновлять `casting/projects.json` и карточку проекта.

---

## 8. Нефункциональные требования

- **Производительность:**  
  - OCC‑анализ не должен запускаться повторно без изменения STEP‑файла;
  - UI‑изменения параметров (тип, материал, усадка, припуски) должны отрабатывать без заметной задержки.

- **Надёжность:**  
  - при ошибках OCC или некорректных STEP‑файлах показывать понятные текстовые ошибки;
  - не допускать падения Streamlit/React из‑за кастомного JS.

- **UX:**  
  - интерфейс должен быть визуально согласован с текущими 3D‑проектами;
  - названия параметров и подсказок — на русском, без жаргона.

---

## 9. Принятие работы

- На странице «Литьё» отображается список проектов с колонками **Название / Материал / Стоимость / Дата**, сортировка по активности.
- Кнопка **«Новое литьё»** создаёт проект, открывается страница проекта с аплоудером STEP.
- После загрузки STEP:
  - отображается 3D‑вид слева и параметры справа;
  - рассчитаны объём, масса, габариты и min толщина стенки;
  - можно выбрать тип литья, материал, задать усадку и припуски.
- Стоимость литья пересчитывается при изменении параметров и отображается и на карточке, и в списке проектов.
- При слишком малой толщине стенки для выбранного типа литья показывается предупреждение.
