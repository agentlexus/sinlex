# ТЗ: классификация токарной обработки и тела вращения (STEP)

Версия: **2026-06-02**  
Статус: **внедрено (2026-06-02)** (сформировано по результатам анализа, без внедрения)  
Связанный код: `extraction_tool/extractor.py`  
Эталонный кейс: `projects/ротор/FRM.698-00A_Корзина_для_Р-4.STEP/`

---

## 1. Цель

Исключить **ложное назначение «Токарная»** для деталей, которые **не являются телом вращения** (корпуса, корзины, листовые и крупногабаритные фрезерные детали), и одновременно **сохранить корректную токарку** для прутков, валов, дисков и крупных поковок/отливок с явным вращательным наружным контуром.

Ключевое требование: решение о токарке и о «теле вращения» должно опираться на **разделение наружных и внутренних цилиндрических граней** (`outer` / `bore`), а не только на AABB и общее число цилиндров.

---

## 2. Проблема (as-is)

### 2.1. Симптом на FRM.698 «Корзина для Р-4»

| Поле | Фактическое значение | Ожидание |
|------|----------------------|----------|
| `operations` | `["Токарная", "Фрезерная (5-осевая)"]` | без «Токарная» (или только фрезерные процессы) |
| `part_family` | `oversize` | корректно |
| `model_size.format` | `rod` (Ø≈552, L≈240) | не трактовать как пруток |
| `cyl_face_count` | 164 | — |
| `elongation_index` | ~2.3 | недостаточно для «прутка» |
| `workpiece.type` | «Плита» при `format=rod` | согласовать с геометрией |

**Причина:** `_revolution_geometry_profile()` выставляет `rotational=True` при «круглом» AABB (`cross_round`) и большом числе цилиндров **без** учёта того, что большинство цилиндров — отверстия, фаски, рёбра, а не коаксиальный наружный контур. `_infer_processes()` добавляет «Токарная» при `rotational` или `hybrid_turn_mill`.

### 2.2. Разделение outer/bore уже есть, но не везде используется

| Механизм | Функция | Результат |
|----------|---------|-----------|
| Геометрия P±ε·n | `_cylindrical_shell_kind()` | `outer`, `bore`, `ambiguous` |
| Сбор записей | `_collect_cylindrical_face_records()` | `kind`, `d`, `area`, ось |
| Пруток (rod) | `_analyze_rod_family()` | outer → валы, bore → отверстия |
| Не-rod отверстия | `_holes_from_bore_clustering()` | только `bore` |
| Установы | `_collect_setup_face_elements()` | `cyl_kind` |
| **Вращение / токарка** | `_revolution_geometry_profile()`, `_infer_processes()` | **не используют `kind`** |
| Быстрый путь валов | `_face_details()` | порог Ø (`_cyl_hole_max_diameter`), **без** `kind` |

Для `part_family != "rod"` (в т.ч. `oversize`) полный разбор `_analyze_rod_family()` **не вызывается**; токарность определяется слабым профилем вращения по AABB.

## 3. Область работ

### 3.1. В scope

- Расчёт метрики **`rotation_confidence`** (`rot_conf`, 0…1) по наружным цилиндрам, осям, площадям.
- Пересмотр условий **`rotational`** и списка **`operations`** (процесс «Токарная»).
- Правила **пруток из проката** (Ø ≤ 300 мм, L/D) vs **крупная поковка/отливка** (Ø > 300 мм, высокий `rot_conf`).
- Согласование **`workpiece`**, **`model_size.format`**, **`part_family`** для `oversize` / `plate`.
- Диагностические поля в JSON анализа (для отладки и UI).
- Регрессионные STEP-кейсы (см. §8).

### 3.2. Вне scope (отдельные ТЗ)

- Изменение тарифов, LLM-промптов, расчёта стоимости по операциям.
- Полная переработка `count_setups_from_shape` (допустима точечная подстройка при наличии `rot_conf`).
- Замена `fast_analyze` на полный ray-casting стенок (уже отдельная политика ≥512 КБ).

## 4. Требования (to-be)

### 4.1. Обязательное использование outer/bore для вращения

1. Ввести единую функцию сбора цилиндрического профиля, например `_rotation_shell_profile(shape, bbox) -> dict`, на базе `_collect_cylindrical_face_records()` (и при необходимости `_collect_cone_face_records()` для наружных конусов).
2. Все решения о **`rotational`** и **«Токарная»** для семейств `plate`, `oversize`, `hybrid_shaft`, `impeller` (где применимо) — только после расчёта `rot_conf` и долей площади **наружной** оболочки.
3. **`_face_details()`** для списков `shafts` / первичного роутинга: либо вызывать `_cylindrical_shell_kind` для цилиндров, либо не использовать `shafts` из `_face_details` в `_revolution_geometry_profile` (предпочтительно — отдельный профиль из §4.2).

### 4.2. Метрика `rotation_confidence` (`rot_conf`)

Рассчитывается в диапазоне **[0.0, 1.0]** и сохраняется в результате анализа.

**Входные признаки (минимальный набор):**

| Признак | Описание |
|---------|----------|
| `outer_cyl_area_share` | Σ area(`outer` ∪ допустимый `ambiguous`) / Σ area(все цилиндры) |
| `bore_cyl_area_share` | Σ area(`bore`) / Σ area(все цилиндры) |
| `main_axis_coaxiality` | Доля наружных цилиндров с осью в пределах угла ≤8° к главной оси (как в `_infer_main_axis_from_cylinders`) |
| `outer_diameter_span` | (max Ø_outer − min Ø_outer) / max Ø_outer по наружным кластерам |
| `plane_penalty` | Штраф при высокой доле плоских граней, не перпендикулярных главной оси (боковые плоскости корпуса) |
| `bbox_cross_round` | Текущий `cross_round` из `_bbox_axis_profile` — **вспомогательный**, не достаточный сам по себе |

**Рекомендуемая логика (уточняется при реализации):**

- Базовый вклад: `outer_cyl_area_share` × `main_axis_coaxiality`.
- Штраф: высокий `bore_cyl_area_share` при низком `outer_cyl_area_share`.
- Штраф: `plane_penalty` > порога (много боковых плоскостей при низкой коаксиальности наружных цилиндров).
- Штраф: `outer_diameter_span` > 0.35 (нет единого наружного диаметра — не вал).

**Обработка `ambiguous`:**

- Не считать автоматически наружными.
- Относить к `outer`, только если Ø ≥ 0.55×Ø тела (как в `_analyze_rod_family`) **и** ось коаксиальна главной; иначе к `bore` при Ø ≤ 0.92×Ø тела; иначе не учитывать в `rot_conf` (или учитывать с весом 0.5).

### 4.3. Правила назначения «Токарная»

**Общий запрет:** если `rot_conf < 0.60` → **не добавлять** «Токарная» (исключение: явный флаг `hybrid_turn_mill` по §4.3.4).

#### 4.3.1. Пруток / прокат (Case A)

Условия **все одновременно**:

- Чистовой наружный Ø (по наружным цилиндрам или AABB поперечника) **≤ 300 мм**
- **L/D ≥ 1.8** (длина / Ø по главной оси)
- **`rot_conf ≥ 0.75`**
- `part_family` после роутинга — `rod` **или** базовая геометрия удовлетворяет `is_elongated_rod` с подтверждённым `outer_cyl_area_share`

Заготовка: **«Пруток»** (`workpiece`), `model_size.format = "rod"`.

#### 4.3.2. Крупная поковка / отливка с токаркой (Case B)

- Наружный Ø **> 300 мм**
- **`rot_conf ≥ 0.90`**
- `outer_cyl_area_share ≥ 0.25` (порог уточнить по регрессии)
- Не срабатывает `plane_penalty` как у корпусной корзины

Заготовка: **«Поковка»** / «Отливка» (новый тип или подтип в `workpiece`) — **не** «Пруток» при Ø > 300.

#### 4.3.3. Диск / фланец (Case C)

- `is_disc` из `_bbox_axis_profile`
- **`rot_conf ≥ 0.80`**
- Наружные цилиндры коаксиальны, `plane` умеренно

Токарная + при необходимости 3-осевая фрезерная (текущая логика `rod_needs_milling` сохраняется).

#### 4.3.4. Гибрид токарка+фреза (Case D)

- Существующий `_is_hybrid_turn_mill_body()` — сохранить, но **не** выставлять `rotational=True` только из-за `cross_round=False` и длины.
- «Токарная» допустима при `hybrid_turn_mill=True` и `rot_conf ≥ 0.55` **или** по явным ключевым словам в имени (намотка).

#### 4.3.5. Oversize / plate без вращения

- `part_family == "oversize"` и `rot_conf < 0.60` → **без** «Токарная»
- Допускается «Фрезерная (5-осевая)» по текущим правилам `_needs_5axis_milling`
- `workpiece.type` = **«Плита»** или **«Блок»** (не «Пруток»), `format` = `"box"` / `"plate"`

### 4.4. Изменение `_revolution_geometry_profile`

- Поле `rotational` выставлять **только** при выполнении правил §4.3 (через `rot_conf` и Case A–D), а не по формулам `classic_rev` / `compact_rev` с одним `cross_round`.
- Старые эвристики (`classic_rev`, `compact_rev`, `simple_blank`, `elongated_shaft`) — либо удалить, либо оставить как **внутренние** подсказки с весом ≤0.3 к `rot_conf`, но **не** как прямой OR к `rotational`.

### 4.5. Согласование семейств и заготовки

| Условие | `part_family` | `workpiece` | `format` |
|---------|---------------|-------------|----------|
| Ø > 400 мм или масса > 100 кг | `oversize` (как сейчас) | не «Пруток» без Case B | `box` / `plate` |
| Case A | `rod` | Пруток | `rod` |
| Case B | `oversize` или отдельный тип* | Поковка/отливка | по геометрии |
| Корзина FRM.698 | `oversize` | Плита/блок | `box` |

\* Для Ø > 300 не использовать семейство `rod` только из AABB.

### 4.6. Производительность

- `_collect_cylindrical_face_records()` вызывать **один раз** за анализ; кэшировать в локальной переменной `extract_step` / `analyze`.
- Для `fast=True` (STEP ≥ 512 КБ): **разрешено** считать `rot_conf` (классификатор P±ε·n на цилиндрах — приемлемая стоимость относительно полного ray-casting стенок).
- Не дублировать обход всех граней: объединить с существующим проходом, где возможно.

## 9. Риски и открытые вопросы

1. Порог **300 мм** — бизнес-правило проката.
2. Веса `rot_conf` — калибровка на 10–15 STEP через `config.py`.
3. Частичный вращательный пояс на корпусе — Case B не должен срабатывать.
4. Ключевые слова в имени (`намотк`) — override, не основной критерий.

---

## 10. История решений (анализы 2026-06-02)

1. **outer/bore** (`_cylindrical_shell_kind`) — критично для токарки на **всех** семействах.
2. AABB `cross_round` + число цилиндров — недостаточны (FRM.698).
3. Пруток: **Ø ≤ 300 мм**, **L/D ≥ 1.8**, **`rot_conf ≥ 0.75`**.
4. Крупное вращение: **Ø > 300 мм**, **`rot_conf ≥ 0.90`**.
5. Нижняя отсечка: **`rot_conf < 0.60`** → без токарки (кроме hybrid по §4.3.4).
