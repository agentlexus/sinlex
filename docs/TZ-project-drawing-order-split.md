## 1. Цель

После загрузки и анализа STEP, в нижней части страницы проекта (текущий блок **«Анализ чертежа»**) разделить область на **две равные колонки**:

| Левая половина | Правая половина |
|----------------|-----------------|
| Анализ чертежа: загрузка (вверху страницы), кнопки **«Анализировать»** / **«Поток»**, статусы, уведомление о готовности | **«Размещение заказа»** — заглушка (кнопка без бизнес-логики) |

Главное UX-изменение: **текст результата анализа чертежа не показывать inline на синем фоне** (`st.info`). Вместо этого — компактное уведомление в левой колонке и кнопка **«Показать»**, открывающая **модальное окно** с полным текстом поверх основного модуля.

Бэкенд анализа (API, кэш, критерии стоимости, гибрид) **не меняется**.

### 3.2. Левая колонка — состояния UI

| Состояние | Содержимое |
|-----------|------------|
| **A. Нет PDF** | `st.info` / caption: «Загрузите чертёж **вверху** (справа от STEP)…» — как сейчас, но **только в левой колонке** |
| **B. PDF есть, анализ не запускался** | Баннер scan-risk (если есть кэш), кнопки **«Анализировать»** / **«Поток»**, «Удалить чертёж» |

### 3.3. Модальное окно «Показать»

При нажатии **«Показать»** открывается **диалог поверх страницы** с полным текстом анализа.

**Реализация (Streamlit ≥ 1.33, в проекте 1.57):** `@st.dialog` / `st.dialog`.

Содержимое модалки:
- Заголовок: **«Результат анализа чертежа»** (без упоминания провайдеров LLM — см. README, приватность).
- Тело: `format_analysis_for_display(analysis)` — тот же форматированный markdown, что сейчас в `st.info`.
- Опционально внизу модалки (свернуто или мелким шрифтом):
  - `render_drawing_pages_caption(extraction)`
  - `render_drawing_layout_debug(extraction)` — только если уже включено в текущем окружении (не расширять scope отладки).
- Кнопка закрытия: стандартная «×» диалога + **«Закрыть»** внутри.

**Session:** флаг `drawing_result_dialog_open_{slug}` или передача через `st.session_state` + rerun при клике «Показать».

**Не в модалке:** блок стоимости, 3D-viewer, параметры заготовки — остаются на фоне (затемнение overlay — стандарт Streamlit dialog).

Одинаковое поведение для:
- классического `deep_analysis_{slug}`;
- гибрида `hybrid_result_{slug}` (после unlock / `status=ok`).

### 3.4. Кнопки после успешного анализа

Минимальный вариант (v1):
- Кнопки **«Анализировать»** / **«Поток»** скрыты (как сейчас при `_drawing_analysis_complete_for_ui`).
- Доступны: **«Показать»**, **«Удалить чертёж»**.
- Повторный анализ — только после удаления чертежа или смены PDF (текущая семантика).

Опционально v1.1 (не блокер): ссылка **«Запустить снова»** с подтверждением — отдельная задача.

## 4. Область работ

### 4.1. In scope

1. Обёртка `render_expert_analysis_section` в две колонки (или новая функция-обёртка `render_project_drawing_order_section` в `pdf_analysis.py`, вызываемая из `5_Upload.py`).
2. Убрать inline `st.info` с текстом LLM из основного потока страницы (классика + гибрид).
3. Компактное уведомление + **«Показать»** + `st.dialog`.
4. Правая колонка — заглушка «Размещение заказа».
5. Сохранить: scan-risk, гибрид polling, оплату Потока, удаление чертежа, персист в `data.txt` / `analysis_log.jsonl`.
6. Обновить текст в `2_Projects.py` при расхождении с новым UX (если нужно).

### 4.2. Out of scope

- Реальная логика размещения заказа (ERP, email, CRM).
- Изменение API `/expert-analysis`, `/hybrid-analysis/*`.
- Блок **«Технологическая карта»** и **«Стоимость»** — порядок и содержимое без изменений.
- Модуль литья, страница списка проектов (кроме подсказок).
- Вынос PDF-uploader из верхней строки в нижний блок.

### 5.2. Псевдокод

```python
def render_project_drawing_order_panel(...):
    st.markdown("### …")  # общий заголовок секции
    col_draw, col_order = st.columns(2, gap="large")
    with col_draw:
        render_expert_analysis_section(...)  # без full-width st.info
    with col_order:
        render_order_placement_stub(slug, project_name)


@st.dialog("Результат анализа чертежа", width="large")
def drawing_analysis_dialog(slug: str):
    text = resolve_analysis_text_for_display(slug)  # deep или hybrid
    st.markdown(format_analysis_for_display(text))
    if st.button("Закрыть"):
        st.rerun()
```

Ключи session (префикс по `slug`):
- `drawing_analysis_dialog_{slug}` — bool, открыт ли диалог
- Существующие: `deep_analysis_{slug}`, `hybrid_result_{slug}`, `pdf_scan_hash_{slug}` — **без переименования**

### 5.3. Зависимости от других блоков

- **Техкарта** читает `deep_analysis_{slug}` / hybrid — текст остаётся в session и на диске; скрытие inline-вывода **не** ломает `render_tech_card_section`.
- **Стоимость** — `resolve_drawing_criteria_for_costing` после анализа; пересчёт при успехе **без изменений** (`5_Upload.py` после секции).
- **Сверка чертёж vs STEP** — если выводится внутри `format_analysis_for_display`, остаётся в модалке.

### 5.4. Адаптив (мобильные)

При ширине ≤768 px (как в `app.py`): колонки Streamlit становятся вертикальным стеком — **сначала** анализ чертежа, **затем** размещение заказа. Доп. CSS не обязателен в v1.

## 6. Приёмочные критерии

- [ ] Открыт 3D-проект с проанализированным STEP: нижний блок — **две колонки** 50/50.
- [ ] PDF не загружен: левая — подсказка; правая — заглушка заказа.
- [ ] PDF загружен, анализ не выполнен: в **левой** колонке видны **«Анализировать»** и **«Поток»** (если гибрид включён).
- [ ] После успешного классического анализа: **нет** синего `st.info` с полным текстом на странице; есть уведомление + **«Показать»**.
- [ ] **«Показать»** открывает модальное окно с полным текстом; закрытие возвращает на страницу без потери данных.
- [ ] После успешного **Потока** — то же (уведомление + модалка), без дублирования текста inline.
- [ ] Правая колонка: кнопка **«Размещение заказа»** → toast/сообщение «в разработке», без сетевых запросов.
- [ ] Стоимость и техкарта ниже работают как до изменения.
- [ ] Удаление чертежа сбрасывает состояние уведомления и закрывает доступ к «Показать».
- [ ] Литьё (`8_Casting_Project.py`) визуально **не** изменилось.

## 8. Риски

| Риск | Митигация |
|------|-----------|
| Длинный текст в диалоге — прокрутка | `width="large"`, markdown в scrollable dialog body |
| Двойной rerun при открытии диалога | Один ключ session, debounce на кнопке «Показать» |
| Регрессия критериев стоимости | Smoke: анализ PDF → пересчёт блока «Стоимость» без открытия модалки |
| Пользователь не видит результат без клика | Явный success-текст «Результат сохранён»; техкарта по-прежнему доступна ниже |

## 9. Чеклист тестирования (smoke)

1. Новый проект: STEP → PDF → Анализировать → уведомление, нет синего блока → Показать → текст в модалке.
2. Проект с кэшем: открыть из списка → уведомление + Показать без повторного API.
3. Поток (если `ENABLE_HYBRID_SUFFLER=1`): завершение → модалка, не inline.
4. Удалить чертёж → снова кнопки анализа.
5. «Размещение заказа» → toast, без ошибок в network.
6. Техкарта и стоимость — значения совпадают с веткой до патча.
