# ТЗ: улучшение 3D-viewer (Three.js) в модуле «Литьё»

Версия: **2026-06-05**  
Статус: **внедрено (2026-06-05)**  
Связанные документы: `TZ-casting-module.md`, `TZ-step-upload-limits.md`  
Связанный код:
- `page_modules/8_Casting_Project.py` — страница проекта литья
- `page_modules/viewer_3d.py` — общий Three.js viewer (3D + литьё)

---

## 1. Цель

Повысить удобство и информативность окна **Three.js** на странице литьевого проекта: модель должна быть крупнее и стабильнее, геометрия STEP — визуально связана с viewer, изменение параметров литья справа не должно перезагружать 3D, ошибки загрузки GLB — понятны пользователю.

Базовый viewer **переиспользуется** из `viewer_3d.py`; изменения не должны ломать страницу 3D-проектов (`5_Upload.py`), если не оговорено явно.

---

## 2. Текущее состояние (as-is)

| Аспект | Реализация |
|--------|------------|
| Viewer | `render_3d_viewer()` — общий для 3D и литья, без параметров режима |
| Разметка | `st.columns([0.5, 0.5])`, заголовок `### 3D модель` |
| Высота | 420 px (жёстко в HTML `#wrap` и `st.components.v1.html`) |
| GLB | ≤ 20 МБ — inline base64 (`GLB_INLINE_MAX_BYTES`); иначе fetch `/api/casting/glb/...` |
| Материал | Все mesh → единый серый `MeshStandardMaterial` |
| Управление | Только `OrbitControls` (мышь), без кнопок сброса/видов |
| Подписи геометрии | Блок «Геометрия STEP» **ниже обеих колонок**, не под viewer |
| Перезагрузка | Любое изменение параметров литья → полный `st.rerun` → пересоздание iframe с GLB |
| Embed | `/embed/3d/{project}` — только `projects/`, маршрута для `casting/` нет |
| Streamlit API | `st.components.v1.html` (deprecated; целевой — `st.iframe`) |

Известные кейсы:
- `FRM.698-00A_Корзина_для_Р-4.STEP` — GLB ~1.3 МБ, inline, крупногабарит (~550 мм) — модель мелкая в колонке 50%.
- Крупные STEP — до конвертации GLB возможен 404 на fetch-path; сообщение только внутри iframe.

## 3. Область работ

### 3.1. В scope

1. Параметризация viewer для литья (высота, пропорции колонок).
2. Подписи габаритов/геометрии STEP непосредственно под окном 3D.
3. Изоляция viewer от rerun при изменении параметров литья (`@st.fragment` или эквивалент).
4. Улучшение UX загрузки и отображения ошибок GLB.
5. Панель управления камерой (сброс, стандартные виды).
6. Опционально для этапа 2: wireframe, стилизация контейнера.
7. Миграция `st.components.v1.html` → `st.iframe` (без изменения логики Three.js).
8. Embed-маршрут для литья (открытие viewer в отдельной вкладке).

### 3.2. Вне scope (отдельные ТЗ / эпики)

- Heatmap / подсветка зон тонких стенок по граням GLB.
- Визуализация припуска и усадки в 3D (полупрозрачный stock-bbox, scale модели).
- Изменение OCC-анализа толщины стенок.
- Изменение лимитов STEP/GLB (`TZ-step-upload-limits.md`).
- Полноэкранный viewer внутри Streamlit без embed.

## 5. API и контракты

### 5.1. Сигнатура `render_3d_viewer` (целевая)

```python
def render_3d_viewer(
    project_name: str,
    glb_base64: str = "",
    glb_size: int = 0,
    *,
    height: int = 420,
    mode: str = "default",  # "default" | "casting"
    show_streamlit_errors: bool = True,
) -> None:
    ...
```

### 5.2. GLB для литья

Без изменения контракта API:
- `GET /api/casting/glb/{project_name}?key=...&email=...&folder=...`
- `api_resource_prefix()` → `"casting"` при `project_storage() == "casting"`.

### 5.3. Константы (без изменения лимитов)

| Параметр | Значение | Файл |
|----------|----------|------|
| Inline GLB | 20 МБ | `upload_limits.py` → `GLB_INLINE_MAX_BYTES` |
| Высота viewer (литьё) | 520 px (рекомендация) | `8_Casting_Project.py` |
| Высота viewer (3D) | 420 px | по умолчанию `viewer_3d.py` |

### 5.4. URL embed-viewer (обязательный контракт)

После этапа 6 viewer в Streamlit — это `st.iframe(src)`, где `src` — **внешний URL**, а не inline HTML.

| Слой | Маршрут | Куда попадает запрос |
|------|---------|----------------------|
| nginx | `/api/*` | FastAPI `:8001` (префикс `/api/` срезается) |
| nginx | `/` | Статический **лендинг** sinlex.tech |
| FastAPI | `/embed/3d-viewer/{project}` | HTML Three.js viewer |
| FastAPI | `/embed/3d-casting/{project}` | То же для `casting/` |

**Правильный URL для iframe:**

```text
https://sinlex.tech/api/embed/3d-viewer/{project}?key=...&email=...&folder=...&height=520&storage=casting
https://sinlex.tech/api/embed/3d-casting/{project}?key=...&...
```

**Неправильный URL (симптом: в окне viewer — лендинг):**

```text
https://sinlex.tech/embed/3d-viewer/...
```

Формирование — только через `api_browser_base()` (возвращает `https://{host}/api`), **не** через `site_base_from_browser_api()` (отрезает `/api` и даёт корень сайта).

## 6. Нефункциональные требования

- **Производительность:** fragment viewer не должен увеличивать время первой отрисовки; повторные rerun страницы литья без смены STEP/GLB не передают base64 GLB заново в HTML, если viewer не в зоне rerun.
- **Совместимость:** Chrome/Firefox последних версий; WebGL2 или WebGL1 fallback Three.js r15x (как в `static/three/`).
- **Безопасность:** не расширять передачу `API_KEY` в клиентский JS сверх текущей схемы fetch URL.
- **UX:** русские подписи; согласованность с блоком «Параметры литья» (отступы `gap="large"`).
- **Регрессии:** smoke-тест на `5_Upload.py` и `8_Casting_Project.py` после каждого этапа.

## 11. Постмортем: регрессия «лендинг во viewer» (2026-06-05)

### Суть

После миграции на `st.iframe` (этап 6) и доработки литьевого overlay (этап 7) в окне 3D вместо модели периодически отображался **маркетинговый лендинг** sinlex.tech. API и GLB были исправны — ломался только URL, который Streamlit подставлял в iframe.

### Причина

Функция `viewer_embed_src()` в `page_modules/viewer_3d.py` собирала адрес так:

```python
site = site_base_from_browser_api(api_browser_base())  # https://sinlex.tech
return f"{site}/embed/{embed_path}/{enc_name}?{q}"
```

Запрос шёл на `https://sinlex.tech/embed/...`. В nginx нет `location` для `/embed/` — срабатывает `location /` → отдаётся лендинг. Маршруты FastAPI живут за прокси `/api/` → на бэкенде путь `/embed/...`.

Регрессия возникла **дважды**: сначала при внедрении этапа 6, затем при рефакторинге этапа 7 (`casting_ctx`, query-параметры припуска/усадки), когда снова вернули `site_base_from_browser_api()`.

### Патч

**Файл:** `page_modules/viewer_3d.py`, функция `viewer_embed_src()`.

**Было (неверно):**

```python
site = site_base_from_browser_api(api_browser_base())
return f"{site}/embed/{embed_path}/{enc_name}?{q}"
```

**Стало (верно):**

```python
# nginx: только /api/* → FastAPI; /embed/* без префикса отдаёт лендинг
api_base = api_browser_base().rstrip("/")
return f"{api_base}/embed/{embed_path}/{enc_name}?{q}"
```

Пример итогового URL:

```text
https://sinlex.tech/api/embed/3d-casting/FRM.698-00A_Корзина_для_Р-4.STEP?key=...&email=...&folder=sinlex_admin&height=520&casting=1&allowance_mm=0.0000&shrink_pct=0.0000
```

### Проверка после правки

```bash
# Должен вернуть HTTP 200 и <!DOCTYPE html> с Three.js, не редирект на /
curl -sI "https://sinlex.tech/api/embed/3d-casting/{project}?key=...&email=...&folder=..."

# Локально (FastAPI напрямую)
curl -s "http://127.0.0.1:8001/embed/3d-casting/{project}?..." | head -c 200
```

После деплоя: `sudo systemctl restart sinlex-streamlit` (и `sinlex-server`, если менялся `embed.py`).

### Правило для следующих изменений

1. **Не использовать** `site_base_from_browser_api()` для URL embed-viewer.
2. Любой новый embed-маршрут в FastAPI (`/embed/...`) вызывать из браузера только как **`/api/embed/...`**.
3. При рефакторинге `viewer_embed_src` — smoke-тест: в iframe не лендинг, а спиннер «Загрузка 3D-модели…».
4. `site_base_from_browser_api()` допустим для ссылок на **корень сайта** (оплата, маркетинг), но не для API/embed.

## 12. Патч: режим «Отливка», «Каркас», освещение solid (2026-06-06)

### 12.1. Цель

В панели камеры литьевого viewer (рядом с «Припуск», «Усадка») дать режим просмотра **готовой отливки** — без детали, только заготовка с припуском как **непрозрачное** тело с корректным освещением. Исправить пропадающие грани stock GLB в solid-режиме.

**Файлы:** `page_modules/viewer_3d.py`, `casting_allowance_offset.py`, `casting_stock_glb.py` (см. `TZ-casting-allowance-offset.md` §18).

**Изоляция:** только `CASTING_CTX.enabled` / `isCastingView` — `mode="default"` и 3D-проекты **не затрагиваются**.

### 12.2. UI — кнопки панели камеры (литьё)

| ID | Подпись | Поведение |
|----|---------|-----------|
| `btnStock` | Припуск | Toggle полупрозрачной оболочки stock (`stockShellMat`, opacity ~0.21) |
| `btnShrink` | Усадка | Toggle `model.scale = 1 - shrink_pct/100` |
| `btnCasting` | **Отливка** | Toggle solid-режима заготовки (см. §12.3) |
| `btnWireframe` | **Каркас** | В активном состоянии подпись **Тело** (wireframe off) |

Кнопки литья вставляются через `addCastingBtn()` в начало `#camBar` (как «Припуск» / «Усадка»).

### 12.3. Режим «Отливка» (`castingViewOn`)

**Включено:**
- `model.visible = false` — mesh детали скрыт;
- stock принудительно загружается (`stockOn = true`, кнопка «Припуск» active);
- материал stock: `castingSolidMat` — `MeshStandardMaterial`, цвет `#14b8a6`, `metalness: 0.22`, `roughness: 0.38`, **`side: THREE.DoubleSide`**;
- `stockGroup.renderOrder = 0`; рёбра bbox-fallback скрыты;
- overlay: дополнение `Отливка: солид`.

**Выключено:**
- `model.visible = true`;
- stock снова `stockShellMat` (полупрозрачный), если «Припуск» включён.

**Ограничения:**
- при `allowance_mm <= 0` включение «Отливка» игнорируется;
- при выключении «Припуск» режим «Отливка» сбрасывается.

### 12.4. Материалы и свет

| Элемент | Материал | Примечание |
|---------|----------|------------|
| Деталь (литьё) | `MeshStandardMaterial`, `#b8c4c0` | Без изменений |
| Припуск (overlay) | `MeshBasicMaterial`, бирюза, opacity 0.21 | Без освещения — намеренно |
| Отливка (solid) | `MeshStandardMaterial`, бирюза `#14b8a6` | С освещением сцены (ambient + hemisphere + key/fill/rim, ACES) |

**Не использовать** `MeshBasicMaterial` для solid «Отливка» — даёт плоский однотонный цвет без градиентов.

### 12.5. История исправлений рендера (контекст патча)

| Симптом | Причина | Решение |
|---------|---------|---------|
| Чёрное тело | `MeshStandardMaterial` + перевёрнутые нормали stock | Правка winding на сервере (§18 offset ТЗ); `DoubleSide` в viewer |
| Плоский цвет | Временный `MeshBasicMaterial` для solid | Вернуть `MeshStandardMaterial` |
| Пропадающие грани | `FrontSide` + неверный winding; пропуски OCC triangulation; эвристика «от центра bbox» на вогнутых участках | `Face.Orientation()` при экспорте; multi-pass mesh; убрать клиентский flip по центроиду; `DoubleSide` |

### 12.6. Критерии приёмки

- [x] «Отливка»: деталь не видна, заготовка — непрозрачная бирюза **с** бликами/градиентами.
- [x] Повторное нажатие «Отливка» возвращает обычный вид.
- [x] Нет «дыр» на вогнутых участках (Корзина, припуск 5 мм) после пересборки stock GLB v3.
- [x] Wireframe: подписи **Каркас** / **Тело**.
- [x] `5_Upload.py` / `embed/3d-viewer`: без кнопок «Отливка», «Припуск», «Усадка».

### 12.7. Деплой

```bash
sudo systemctl restart sinlex-server sinlex-streamlit
```

После смены `STOCK_OFFSET_ALGO_VERSION` (v3) — перегенерация stock GLB при следующем запросе или лёгкое изменение `allowance_mm` в UI.
